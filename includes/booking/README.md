# سیستم نوبت‌گیری - Medical Records

## مقدمه
این سیستم نوبت‌گیری برای افزونهٔ Medical Records ایجاد شده است و اجازه می‌دهد تا بیماران نوبت از پزشکان گرفته و پزشکان بتوانند ساعات کاری‌شان را مدیریت کنند.

---

## فایل‌های اصلی

### 📁 پوشهٔ `includes/booking/`

1. **class-booking.php**
   - کلاس اصلی `MR_Booking` برای مدیریت نوبت‌ها
   - توابع دریافت و ذخیرهٔ تنظیمات دکتر
   - توابع محاسبهٔ قیمت‌ها
   - توابع دریافت ساعات موجود

2. **shortcode-booking.php**
   - شورتکد `[medical_booking]` برای بیماران
   - پنج مرحلهٔ نوبت‌گیری:
     - **مرحله ۱**: انتخاب دکتر و نوع ملاقات (حضوری/انلاین)
     - **مرحله ۲**: انتخاب تاریخ و ساعت
     - **مرحله ۳**: وارد کردن شماره موبایل و اطلاعات فردی
     - **مرحله ۴**: بررسی و وارد کردن کد تخفیف
     - **مرحله ۵**: تایید نوبت

3. **doctor-settings.php**
   - تابع `mr_doctor_booking_settings()` برای تنظیماتی دکتر
   - مدیریت ساعات کاری
   - تعیین روزهای کاری
   - تنظیم روزهای تعطیل
   - تعریف قیمت‌های خدمات

---

## ویژگی‌های سیستم

### 🏥 برای بیماران

```
[medical_booking]
```

#### مراحل نوبت‌گیری:

**مرحله ۱: انتخاب دکتر و نوع ملاقات**
- انتخاب دکتر از لیست
- انتخاب نوع ملاقات:
  - **حضوری**: با قیمت مشخص
  - **انلاین**: با گزینه‌های:
    - تصویری (10/20/30/40 دقیقه)
    - صوتی (10/20/30/40 دقیقه)
    - متنی (10/20/30/40 دقیقه)
    - تلفنی (10/20/30/40 دقیقه)
- نمایش قیمت برای هر انتخاب

**مرحله ۲: انتخاب تاریخ و ساعت**
- انتخاب تاریخ
- نمایش ساعات موجود بر اساس ساعات کاری دکتر
- انتخاب یکی از ساعات موجود

**مرحله ۳: اطلاعات تماس و فردی**
- شماره موبایل (الزامی)
- نام (از پروفایل بیمار، قابلٔ ویرایش)
- نام خانوادگی (از پروفایل بیمار، قابلٔ ویرایش)
- تاریخ تولد (الزامی)

**مرحله ۴: خلاصهٔ و پرداخت**
- نمایش خلاصهٔ نوبت
- وارد کردن کد تخفیف (اختیاری)
- نمایش قیمت نهایی
- دکمهٔ پرداخت

**مرحله ۵: تایید موفق**
- پیام تایید
- اطلاع به بیمار برای بررسی بعدی

---

### ⚙️ برای پزشکان

ترتیبات نوبت‌گیری در داشبورد پزشک:

#### **ساعات کاری**
- ساعت شروع (پیش‌فرض: 09:00)
- ساعت پایان (پیش‌فرض: 17:00)

#### **روزهای کاری**
- انتخاب روزهای کاریی (شنبه تا جمعه)
- پیش‌فرض: شنبه تا چهارشنبه

#### **روزهای تعطیل**
- اضافه کردن تاریخ‌های تعطیل
- نوبت‌گیری در این روزها غیرممکن است

#### **ملاقات حضوری**
- فعال/غیرفعال کردن
- تعیین قیمت (تومان)

#### **ملاقات انلاین**
- فعال/غیرفعال کردن
- برای هر نوع (تصویری، صوتی، متنی، تلفنی):
  - فعال/غیرفعال کردن نوع
  - تعیین قیمت برای هر مدت‌زمان:
    - 10 دقیقه
    - 20 دقیقه
    - 30 دقیقه
    - 40 دقیقه

## استفاده از توابع اساسی

### دریافت تنظیمات دکتر
```php
$settings = MR_Booking::get_doctor_settings($doctor_id);
```

### ذخیرهٔ تنظیمات دکتر
```php
MR_Booking::save_doctor_settings($doctor_id, $settings);
```

### دریافت نوبت‌های رزرو شده
```php
$appointments = MR_Booking::get_doctor_appointments($doctor_id, $date);
```

### دریافت ساعات موجود
```php
$available_times = MR_Booking::get_available_times($doctor_id, $date, $duration);
```

### محاسبهٔ قیمت نهایی
```php
$final_price = MR_Booking::calculate_final_price($base_price, $discount_code);
```

## AJAX Handlers

### `mr_get_doctor_services`
- دریافت خدمات پزشک
- پارامترها: `doctor_id`, `_wpnonce`
- برگشت: خدمات و قیمت‌های موجود

### `mr_save_doctor_booking_settings`
- ذخیرهٔ تنظیمات نوبت‌گیری
- پارامترها: فیلد‌های فرم، `_wpnonce`
- نیازمند نقش: `editor` یا `administrator`

## نکات مهم

1. **موقتی**: سیستم فعلی بصورت `Session` بر روی هر مرحله داده‌ها را ذخیره می‌کند
2. **نوبت‌های تضاد**: اگر دو بیمار در یک زمان نوبت گیرند، نیاز به راہ حل وجود دارد
3. **کد تخفیف**: به شدت نیازمند پیاده‌سازی
4. **پرداخت**: موقتاً پیام موفقیت نشان می‌دهد، نیاز به اتصال درگاه واقعی
